# 우쿨렐레 운지 코치 고도화 기술 적용 바이브코딩 프롬프트

> **목표**
> 현재 웹앱에 아직 적용되지 않았거나 부분 적용된 네 가지 고도화 기술을 단계적으로 적용한다.
>
> 1. Landmark 기반 press/lift 분류기
> 2. 시간적 스무딩 고도화
> 3. 카메라 배치 가이드
> 4. 운지 문법 prior
>
> 현재 앱은 이미 `MediaPipe HandLandmarker -> 손끝 좌표 -> homography 지판 좌표 -> 목표 코드 직접 비교` 구조를 갖고 있다. 이 문서는 그 구조를 깨지 않고, 각 기술을 켜고 끌 수 있는 모듈로 추가하는 실행 프롬프트다.

## 0. AI 코딩 에이전트에게 주는 역할 지시

```text
너는 시니어 프론트엔드 + 컴퓨터비전 엔지니어다.
이 저장소는 바닐라 JS 정적 웹앱이며, 빌드 도구 없이 브라우저에서 실행된다.

절대 규칙:
- 기존 동작을 깨지 마라.
- 서버 호출, 백엔드, 외부 API를 추가하지 마라.
- 새 런타임 의존성을 추가하지 마라.
- 모든 카메라/랜드마크 처리는 브라우저 안에서만 끝낸다.
- 모든 고도화 기능은 config 플래그로 개별 ON/OFF 가능해야 한다.
- 채점 정직성 규칙을 지켜라.
  - 스무딩과 prior는 노이즈 완화와 표시 보조용이다.
  - prior로 보정한 결과를 정답 채점에 직접 쓰지 마라.
  - 신뢰도나 카메라 품질이 낮으면 정답/오답을 추측하지 말고 "보류"를 표시하라.

현재 주요 파일:
- index.html: UI
- style.css: 스타일
- main.js: 카메라, MediaPipe 루프, 캘리브레이션, 화면 렌더링
- src/chords.js: 코드 사전
- src/fretboard.js: 지판 좌표 변환
- src/homography.js: 4점 homography
- src/evaluation.js: 목표 코드와 현재 손가락 비교

- tests/domain.test.js: 현재 도메인 테스트

개발 방식:
- 각 Phase는 테스트를 먼저 작성한다.
- 테스트 실패를 확인한 뒤 구현한다.
- 구현 후 `npm test`를 통과시킨다.
- UI가 바뀌는 Phase는 `npm run dev`로 로컬 서버를 띄우고 `npm run test:browser`까지 확인한다.
```

## 5. Phase 1 - Landmark 기반 press/lift 분류기

> 우쿨렐레는 목관처럼 구멍을 직접 막는 악기가 아니다. 이 앱의 press/lift는 "손끝이 현재 지판 셀을 실제로 누르는 상태에 가까운가"를 확률로 표현한다. 최종 채점은 여전히 줄/프렛 위치가 기준이며, press probability는 낮은 신뢰도 보류와 표시 안정화에 쓴다.

### 설계

`src/fingerClassifier.js`를 만든다.

필수 export:

```js
export function normalizeHand(landmarks) {}
export function extractFingerFeatures({ landmarks, finger, pixel, fretboard, string, fret, calibration, canvas }) {}
export class FretPressClassifier {
  constructor(config = DEFAULT_VISION_CONFIG.classifier) {}
  classify(observation, context) {}
}
```

### 특징값

1. `cellCenterScore`
   - 손끝이 판정된 줄/프렛 셀 중심에 가까울수록 높다.
   - `getTargetFretboardPoint({ string, fret })`를 재사용한다.
   - 셀 중심과 현재 `(u, v)` 거리로 `0..1` 점수를 만든다.

2. `curlScore`
   - `posture.js`의 관절 각도를 재사용한다.
   - 손가락이 완전히 펴져 있으면 낮고, 자연스럽게 굽혀져 있으면 높다.
   - 너무 납작하면 `posture.risk`로 감점한다.

3. `depthScore`
   - MediaPipe `z`는 노이즈가 크므로 가중치를 낮게 둔다.
   - `tip.z - mcp.z`의 절댓값과 방향을 보조 정보로만 쓴다.

4. `inFretboardScore`
   - 지판 밖이면 0.
   - 줄/프렛이 null이면 0.

5. `pressProbability`
   - 위 특징의 가중합을 sigmoid 또는 clamp로 `0..1` 변환.

### 상태 판정

```js
if (prob >= pressAbove) state = "PRESS";
else if (prob <= liftBelow) state = "LIFT";
else state = "UNKNOWN";
```

## 6. Phase 2 - 시간적 스무딩 고도화

### 목표

현재 `MovingAverageSmoother`와 `StableEvaluationGate`를 유지하면서, 더 앞단에 다음 세 단계를 추가한다.

1. 랜드마크 좌표 One Euro Filter
2. press probability EMA
3. press/lift 상태 히스테리시스 디바운서

### 새 모듈

`src/temporalFilters.js`

필수 export:

```js
export class LowPassFilter {}
export class OneEuroFilter {}
export class LandmarkFilterBank {}
export class ProbabilitySmoother {}
export class StateDebouncer {}
```

### 복붙 프롬프트

```text
Phase 2를 구현해줘. 목표는 시간적 스무딩을 좌표, 확률, 상태 세 층으로 분리하는 것이다.

파일:
- 추가: src/temporalFilters.js
- 수정: tests/vision-upgrades.test.js
- 수정: src/stability.js는 기존 API를 깨지 말 것. 필요하면 새 파일에서만 구현.

TDD 순서:
1. OneEuroFilter 테스트
   - 첫 샘플은 그대로 반환한다.
   - 같은 값이 반복되면 출력이 같은 값에 수렴한다.
   - 급격한 점프 입력에서는 일반 lowpass보다 적응적으로 따라가야 한다.
2. LandmarkFilterBank 테스트
   - key가 `handIndex-landmarkIndex-axis` 단위로 분리되어야 한다.
   - 다른 손의 같은 landmark가 서로 영향을 주면 안 된다.
3. ProbabilitySmoother 테스트
   - 같은 finger key에 대해 EMA가 적용되어야 한다.
   - reset(key)와 clear()가 동작해야 한다.
4. StateDebouncer 테스트
   - LIFT 상태에서 p가 pressHigh보다 커진 상태가 minFrames 지속되어야 PRESS로 바뀐다.
   - PRESS 상태에서 p가 liftLow보다 낮아진 상태가 minFrames 지속되어야 LIFT로 바뀐다.
   - 중간 확률에서는 기존 상태를 유지한다.

구현:
- OneEuroFilter 공식:
  - alpha(cutoff, dt) = 1 / (1 + tau / dt)
  - tau = 1 / (2 * Math.PI * cutoff)
  - dx = (x - xPrev) / dt
  - edx = lowpass(dx, alpha(dCutoff, dt))
  - cutoff = minCutoff + beta * Math.abs(edx)
  - output = lowpass(x, alpha(cutoff, dt))
- timestamp는 ms로 들어오므로 내부 dt는 초 단위로 바꾼다.
- dt가 0 이하이면 이전 값을 반환한다.
- LandmarkFilterBank.filterHands(hands, t)는 hands 구조를 복사해 landmarks만 필터링한다.
- ProbabilitySmoother.smooth(key, p)는 pSmooth를 반환한다.
- StateDebouncer.update(key, pSmooth)는 `{ state, candidate, frames, changed }`를 반환한다.

main.js 연결:
- detectFingerPositions() 앞에서 MediaPipe landmarks를 LandmarkFilterBank로 필터링한다.
- FretPressClassifier가 낸 pressProbability를 ProbabilitySmoother에 통과시킨다.
- pSmooth를 StateDebouncer에 넣어 pressState를 안정화한다.
- 디버그 패널에 raw probability, smoothed probability, debounced state를 표시한다.
- 기존 MovingAverageSmoother는 pixel 보조 스무딩으로 유지해도 된다. 단, 중복 지연이 과하면 windowSize를 4로 낮춘다.

완료 기준:
```

## 9. Phase 5 - 운지 문법 prior

### 핵심 원칙

prior는 학생을 "정답으로 끌어당기는" 장치가 아니다.

사용 목적:

- 표시용 손가락 위치 흔들림 보정
- 가능한 코드/보이싱 후보의 애매성 표시
- 디버그와 연습 리포트에서 어떤 코드와 혼동되는지 설명

금지:

- prior 결과를 `evaluateVoicing()` 채점 입력으로 사용
- 목표 코드에 가중치를 줘서 오답을 정답처럼 보이게 만들기
- 품질이 낮은 상태에서 prior로 억지 확정

### 새 모듈

`src/fingeringGrammar.js`

필수 export:

```js
export function buildValidVoicingDictionary(chords) {}
export function scoreObservationAgainstVoicing(observations, chord, options = {}) {}
export class FingeringGrammar {
  constructor(dictionary, config = DEFAULT_VISION_CONFIG.grammar) {}
  mapSnap(observations, context = {}) {}
}
```

### MAP score 설계

각 코드 후보에 대해 로그 점수를 계산한다.

```text
score = log(prior)
      + required note likelihood
      + extra penalty
      + physical constraint bonus/penalty
      + confidence bonus
```

세부 규칙:

1. required note likelihood
   - 목표 `(string, fret)` 근처에 관측 손가락이 있으면 가산
   - strictFinger가 켜져 있으면 finger까지 맞을 때 더 크게 가산
   - pressState가 PRESS면 가산
   - pressState가 UNKNOWN이면 작게 가산 또는 보류

2. extra penalty
   - 목표 코드에 없는 위치를 누르면 감점
   - 단, 관측 confidence가 낮으면 감점 약화

3. physical constraints
   - 같은 셀에 여러 손가락 중복 감점
   - 최대 프렛 스팬 초과 감점
   - 손가락 번호 순서가 비현실적이면 감점
   - 바레 가능성은 현재 앱의 단순 모델을 유지하되, 동일 finger+fret이 연속 줄에 쓰인 경우 허용

4. ambiguity
   - 1등과 2등 posterior 차이가 `ambiguityGap`보다 작으면 ambiguous=true
   - UI에는 "C와 C7이 비슷하게 감지됩니다"처럼 표시

### 복붙 프롬프트


## 14. 개발자가 실수하기 쉬운 지점

### 14.1 prior를 채점에 쓰는 실수

나쁜 구현:

```js
const snapped = grammar.mapSnap(observedFingers);
const evaluation = evaluateVoicing(snapped.displayFingers, targetChord);
```

좋은 구현:

```js
const grammar = grammarEngine.mapSnap(observedFingers);
const evaluation = evaluateVoicing(observedFingers, targetChord);
```

### 14.2 품질이 낮은데 오답으로 처리하는 실수

나쁜 구현:

```js
if (quality.score < 0.48) {
  showWrong();
}
```

좋은 구현:

```js
if (quality.gateGrading) {
  grading = { match: null, reason: "quality_gate", evaluation: null };
}
```

### 14.3 press/lift를 절대 정답처럼 믿는 실수

press probability는 웹캠 기반 추정값이다. 위치, 각도, 조명에 흔들린다.

좋은 기준:

```text
줄/프렛 위치 = 채점 핵심
pressProbability = 신뢰도와 보류 판단 보조
pressState UNKNOWN = 정답/오답 확정 대신 고정 안내
```

### 14.4 One Euro 필터 timestamp 단위 실수

`performance.now()`는 ms다. One Euro 내부의 dt는 초 단위로 변환해야 한다.

```js
const dt = (tMs - this.lastTimeMs) / 1000;
```

### 14.5 손 좌표 미러링 이중 적용

현재 앱은 video를 CSS로 `scaleX(-1)` 미러링하고, landmark pixel 변환에서 `1 - x`를 적용한다.

새 엔진에서도 이 정책을 하나의 config로만 사용한다.

```js
normalizedLandmarkToPixel(landmark, canvas, state.isMirrored)
```
