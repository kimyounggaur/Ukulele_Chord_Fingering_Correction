# Vision Upgrade QA Checklist

이 체크리스트는 `운지코치_미적용_고도화기술_적용_바이브코딩_프롬프트.md`의 고도화 기술 적용 후 수동 확인용입니다.

## 자동 검증

- [ ] `npm test`가 통과한다.
- [ ] `npm run dev`로 로컬 서버가 열린다.
- [ ] `npm run test:browser`가 통과한다.

## 카메라와 품질 게이트

- [ ] 카메라 시작 전에는 준비도 UI가 `준비도 --`로 보인다.
- [ ] 손을 화면 밖으로 빼면 "손을 화면에 보여주세요" 계열 안내가 나온다.
- [ ] 손과 지판이 너무 작게 보이면 가까이 두라는 안내가 나온다.
- [ ] 손끝이 화면 가장자리에 걸리면 화면 중앙으로 옮기라는 안내가 나온다.
- [ ] 어두운 환경에서는 조명 안내가 나온다.
- [ ] 품질이 낮을 때 정답/오답으로 단정하지 않고 채점 보류 상태가 나온다.

## Press/Lift와 스무딩

- [ ] 디버그 모드에서 각 손가락의 `pressProbability`, `pSmooth`, `pressState`가 보인다.
- [ ] 손끝을 프렛 중앙에 고정하면 `PRESS`로 안정된다.
- [ ] 프렛 경계에서 살짝 떨려도 안내 문구가 프레임마다 깜빡이지 않는다.
- [ ] 손끝 위치가 애매하면 `UNKNOWN` 또는 낮은 신뢰도 안내가 나온다.

## 운지 문법 Prior

- [ ] 디버그 모드에서 `grammar.alternatives`가 보인다.
- [ ] 유사한 코드가 가까운 점수로 감지되면 애매성 안내가 나온다.
- [ ] prior가 켜져 있어도 실제 오답은 오답으로 남는다.
- [ ] 목표 C에서 약지를 1번줄 2프렛에 두면 C로 snap되더라도 교정은 1번줄 3프렛으로 이동하라고 나온다.

## 우쿨렐레 도메인 회귀

- [ ] 교정 문구는 항상 `1번줄`, `4번줄`처럼 번호로 말한다.
- [ ] "굵은 줄", "얇은 줄" 표현이 나오지 않는다.
- [ ] C, Am, C7 같은 한 손가락 코드는 한 음만 맞아도 정확 판정이 된다.
- [ ] F, G처럼 여러 손가락이 필요한 코드는 누락과 extra를 구분한다.
- [ ] 바레형 코드 테스트가 깨지지 않는다.
